⚙️ How It Works

Connection

Uses PeerJS (WebRTC) to establish a direct peer-to-peer connection. The only server involved is the signaling server, which merely helps peers find each other — it never sees your messages.

Encryption

When a guest joins, the host generates an AES-256-GCM key and sends it over the encrypted channel. All subsequent messages are encrypted with this key. Even if someone intercepted the WebRTC traffic, they couldn't read it.

Room Codes

6-character codes (A-Z, 0-9) generated using crypto.getRandomValues() — cryptographically secure. 2.1 billion possible combinations.

Persistence

None. Everything lives in RAM. No localStorage, no IndexedDB, no cookies, no server logs. Close the tab and the room is destroyed. The host disconnecting destroys the room for everyone.

❓ FAQ

Is my chat really private?

Yes. Messages are end-to-end encrypted with AES-256-GCM. They travel directly between peers via WebRTC. No server ever sees your content. When you close the tab, everything is gone — no logs, no traces.

How many people can join a room?

There's no limit. The host can have as many guests as their bandwidth allows. Each guest connects directly to the host via WebRTC.

What's the file size limit?

Files are sent as base64-encoded text messages. Practical limit is about 3-4 MB (which becomes ~4-5 MB after encoding). For code files, this is usually more than enough.

Which file types can I send?

Any file! We support 50+ programming language extensions including .c, .cpp, .java, .py, .js, .ts, .go, .rs, .rb, .swift, .kt, and many more. You can also send .txt, .md, .json, .csv, and any other file.

What happens if the host leaves?

The room is destroyed. All guests are disconnected. Nothing is saved. If the host temporarily loses connection, they can reconnect and guests will follow.

Does it work on mobile?

Yes! MANGAUD is a PWA (Progressive Web App). It works on any device with a modern browser — Android, iOS, Windows, macOS, Linux. You can install it as an app from your browser.

Why is it slow / laggy?

MANGAUD uses P2P, so speed depends on your internet connection and distance to the other peer. If WebRTC fails (firewalls, NAT), the connection may fall back to a relay, which is slower. Try again or check your network.

Can I block someone?

Yes. The host can block any user's IP from the Users panel. Blocked users are immediately kicked and cannot rejoin the room.

What is Read-Only mode?

Read-only mode means only the host can send messages. The host can selectively grant send permission to specific guests. Useful for presentations or announcements.

Can I transfer host role?

Yes. The host can make any guest the new host from the Users panel. The old host becomes a regular guest. The room continues without interruption.

🎛️ Host Controls

Users Panel

Tap the users icon in the chat header to open the panel. See all connected users with their name, IP, join time, and message count.

Room Mode Toggle

Switch between Normal (everyone can send) and Read-only (only host sends) from the Users panel.

Allow Send

In Read-only mode, tap Allow Send next to a user to grant them send permission.

Make Host

Transfer host role to any user. You'll become a regular guest. The room continues with the new host.

Block User

Block a user's IP to kick them and prevent rejoining. Use this for unwanted guests.
